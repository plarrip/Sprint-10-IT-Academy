- Decision Tree Classifier



'min_impurity_decrease' has 0 by default. This means the predictor won't stop creating new nodes until the impurity is 0 (all remaining observations are from the same class). This may be leading the predictor to over-fitting, so by increasing this hyperparameter the over-fit should reduce, but also will the tree size.
The second hyperparamter we'll harness is 'max_depth' which will help us reducing the tree size even further, making the predictor less over-fitted.
